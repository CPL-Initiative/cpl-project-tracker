How They Work Together
⏰ 4 AM — Cowork Routine
Role: Pipeline executor & orchestrator
Data: Uses whatever is committed in the repo (yesterday's live metrics unless GH Actions has already run)
Deploy: Pushes to session branch → requires PR
Value: Runs the Python pipeline, updates KPI history, generates Word reports, handles workplan logic
+
⏰ 6 AM — GitHub Actions
Role: Data fetcher & publisher
Data: Fetches fresh live metrics + Custom Report directly (unrestricted network access)
Deploy: Pushes directly to main → GitHub Pages updates immediately
Value: Ensures the dashboard always has today's real data on it
=
🎯 Live Dashboard
Updated daily by 6 AM PT with fresh CCCCO data and a fully regenerated HTML dashboard pushed directly to GitHub Pages
🔒 Why the Routine can't fetch live data
Claude Code Routines run in Anthropic's cloud sandbox, which has a platform-managed network allowlist — outbound calls to custom hostnames (like the CCCCO Azure API or the Cloudflare Worker) are blocked with a 403 "Host not in allowlist" error. There is currently no setting to add custom hosts to this allowlist. GitHub Actions runs on GitHub's own servers, which have unrestricted outbound network access, so those same API calls succeed.
