### Patterns that worked / things worth remembering

- **Capture-before-redeploy = your rollback.** Before redeploying a shared live
  function, `get_edge_function` the running version and stash it. Deno validates
  syntax at deploy time and **fails closed** (a bad deploy leaves the prior
  version running), but a *semantically* bad deploy still needs a known-good
  artifact to revert to. We kept v13 captured.
- **The backend is SHARED + LIVE.** The dev tab and the production
  map.rccd.edu widget call the *same* `cpl-chat` + *same* tables. A redeploy or a
  content swap is **global**, not sandboxed to the dashboard. Phase 1 (UI) is
  safe because it only *adds* a CORS origin; Phase 2 (content re-point) will flip
  both surfaces at once — that's the gated decision, not the code.
- **Direct-to-Edge-Function, not via the Cloudflare proxy.** `quickstart.js`
  goes through the Worker for a single buffered Claude call; the chat panel
  instead hits the Edge Function **directly** because it needs the function's RAG
  orchestration + **token-by-token SSE**. The DOM/markdown scaffolding transfers
  from `quickstart.js`; the SSE contract is the function's own.
- **Escape-FIRST markdown-lite.** `cpl_chat.js` HTML-escapes the model text
  *before* applying the tiny markdown subset (`**bold**`, `` `code` ``, links,
  bullets), so a crafted answer can't inject markup. Links are constrained to
  `http(s)` + `rel="noopener noreferrer"`. (Same discipline as the XSS-audit
  methodology note.)
- **Smoke-test all 4 modes after a deploy:** general / college-specific /
  topic / college+topic. The function branches on detected college + topic
  keywords, so a deploy can pass syntax yet break one branch.
- **Verify live state before building** — versions move (SKILL.md said v12,
  live was v13, we shipped v14). `list_edge_functions` / `list_tables` are the
  authoritative read.
